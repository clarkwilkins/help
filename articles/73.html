examine returns looks for a specific situation where a return record should normally exist.

  • An item is on an order.
  • Rate 2 > rate 1 indicates an exchange sale.
  • There is no return record present.

Normally, this would appear to be an anomaly which is correctable. Either, we generate a return record, or we set the two quoted rates to the same amount.

However, if neither of these is desirable, -bypass will allow the report to skip over the return, though it will be noted as a bypassed item.
