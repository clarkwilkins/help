Whether you run stockd or stockdpro, it's easier to get going if you allow us to batch import your critical information. We've created a basic data template which you can download here

Locations (sheet 1)

The first sheet is a list of locations.

LOCATIONS
Put the name of the location here. It does not matter what you call a location — only that you recognize it as a specific place. For example, if you have a standard industrial shelving unit with five shelves, you might call it “SHELF A”. Then you can designate the individual shelves as “SHELF A1”, “SHELF A2”,... , “SHELF A5”
ID #
This is only useful if you already have your locations serialized. It can help us to set up the location in the ITEMS table below, but isn't necessary.

Items (sheet 2)

The second sheet is a list of your items. Taking time to do this will make your transition to stockd much easier because you won't have to manually enter your inventory. Don't worry if this table seems simplistic. There are many more options available — the emphasis here is on just getting started.

ITEM NUMBER
The unique ID# that identifies the item in your inventory.1
SKU
The SKU (or “part number”) is generally assigned by the manufacturer of the item. If your item does not have one available, you can use a custom SKU or just put in “NA”. SKUs are used to map a standard part name for all items with the SKU into your data, so an item with the SKU 123 will always show the same item name.
NAME
The item name. We're going to use the pair SKU:NAME to standardize this.
LOCATION
Where this item resides now. The locations should match Sheet 1.
CONDITION
Enter whether the item is good, bad, or part-out only.2
FOR SALE
yes/no field
SERIAL NUMBER
Optional field

Customers (sheet 3)

The third sheet is a list of your clients. If you're just going to use the basic level stockd, you can skip this part.3

ID
The unique ID# that identifies the client.4
NAME
The company/client name.5
ADDRESS
This is a free-form field — meaning you can drop multiple addresses into it.6
CONTACT
This is a free-form field — meaning you can drop multiple contact names, emails, phone numbers, etc. into it.
NOTES
An optional field to add anything else you want to say about the client.
Users (sheet 4)

The fourth sheet can be used if you want us to create a set of users for you.

NAME
The user's name.
EMAIL
Their email address.
SMS
Their mobile number.7
level
This value is a tier used to mediate overrides between users. Higher level users override lower ones.8

If you will out this spreadsheet as much as possible and email it back to us via support@mystock.com, we can usually load your data within a day at no charge. We're here to help!
