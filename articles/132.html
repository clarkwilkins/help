The job metadata can be accessed via +info. It exposes seven fields - four of which can be edited.

job number
The master number of the entire job.
customers + locations
The customer this job is assigned to, with an optional location.1
suggested job state
servicd looks at all tickets and evaluates what it thinks the state of this job should be based on the highest status ticket.2
actual job state
This control lets you set the overall state of the job.3
billable
This control lets you set the billing status of the job.4
job notes
This private field holds notes about the job.5
customer PO
Optional field.

tickets

All tickets on the job are shown here. Click on any one to move there. You can also use +new to add a ticket.

NOTES
  1. Locations are optional.
  2. This is a suggestion that you can override with the actual state control.
  3. This is a descriptor for the state of the entire job. For example, there may not be any tickets that complete the job, but it gets cancelled. We leave the individual tickets alone and cancel the job as a whole.
  4. There are several status values here — defined by servicd.
  5. This field is never shown to clients.
