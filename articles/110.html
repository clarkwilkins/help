This section is where you define the settings for the user on a specific platform.1

group
Your company is assigned a group which can include one servicd company, one stockd company, and one (or more) accountd companies.2
company
This selects the company.3

The remainder of the form is divided into sections for accountd, servicd, and stockd.

state
This determines if the user can access the service. When this is switched off, the user cannot reach this service within this company.
server
Each Simplexable customer is assigned a private server address.4
reply-to
You can configure the email service so the "sender" and "reply-to" addresses to override the user's main settings.5
level
Access levels are 1-9. Higher level users gain access to more privileges and can lock out changes by lower level users.
notes
You can add optional notes about this group of settings.
NOTES
  1. Specific means this user, this company, this service.
  2. accountd allows the use of multiple sets of company accounts.
  3. While this has to be set, even if you don't use accountd, it is only affecting the accountd/company-specific settings.
  4. Be careful with this or you can disconnect from your server.
  5. The default is the user's primary address. Overrides are useful if you want a specific address used in the context of this platform.
