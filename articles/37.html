Email settings provide a default set of addressees for content you send from accountd, servicd, or stockd.

For a defined setting (beginning with "default"), you have:

  • accountd [to]: list of contacts...
  • accountd [cc]: list of contacts...
  • accountd [bcc]: list of contacts...
  • servicd [to]: list of contacts...
  • servicd [cc]: list of contacts...
  • servicd [bcc]: list of contacts...
  • stockd [to]: list of contacts...
  • stockd [cc]: list of contacts...
  • stockd [bcc]: list of contacts...

What you're doing here is defining an email policy that sets up a default set of recipients for traffic from each platform you're using. For example, assume you have a client with a Billing entity, a Service Coordinator, and a Sales Manager. You might set up a pattern like this as default.

  • accountd [to]: Billing
  • accountd [cc]: empty
  • accountd [bcc]: empty
  • servicd [to]: Service Coordinator
  • servicd [cc]: Billing
  • servicd [bcc]: empty
  • stockd [to]: Sales Manager
  • stockd [cc]: Billing
  • stockd [bcc]: empty

You can also do custom settings on a per-location basis. This is useful when a contact is relevant for location A, but not for location B. Set up a profile specific to each location, and that's the default setting shown under the +email tool.

By default, all contacts with email addresses are shown in the management view, but most are exclude from use via the na option.

It's important, as well, to realize the actual email list for anything being sent can be modified at time of send. What you're defining here is a template policy (starting point) which may be changed as need arises.
