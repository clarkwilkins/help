The setup area is where you choose the user and the settings you want to configure. Users here are considered active/inactive in a global context.1

The first option is basic user configuration as discussed here.

After that, a service option is shown for each registered company and all available services. You can leave a user/service area unconfigured if you are not using it or the user does not need access. You can also go into a particular service to configure the user-specific settings.

  1. A user is defined globally as a part of your group. This being a global-scope tool, the results are filtered on whether the user is active overall — not whether they are active in this platform, for this company. That's a filtering distinction that's relevant in the platform, but not here in the global editor.
