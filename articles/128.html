A “job” is a collection of one or more tickets assigned to a specific customer and service person.

The basics view is most often in use. This is where the fundamental aspects of a job ticket are managed.

job and ticket
Each job has a master number and at least one ticket.1
customer account
Every job is assigned to a customer account at creation. This cannot be changed after dispatch, and all subsequent tickets inherit this assignment.
followup
This is an optional field used to create a reminder to take action on a job ticket at a future time.
job description
This is initially set at dispatch time as the reason for the job. It can change as needed, and each ticket can modify this field independently of the others.
status
The current state of this job ticket.2
assigned
Each ticket is assigned to a specific user.3
arrival
When the assigned user began work on this ticket.
problem fixed
An optional time field.
work finished
When the assigned user finished work on this ticket.
labor and travel
Optional fields to record time spent.4
work performed
A record of the work done on this ticket.
notes
An optional field for adding contextual notes that are not visible to the customer.

other controls

+email
This invokes the email handler, so you can send the service ticket.
+pdf
This generates a PDF of the current service ticket.
+info
This switches you to the job metadata view.
+items
This switches you to the job items view.
+billing
This switches you to the job billing view.
add another ticket
This button switches you to the new ticket tool.
generate a job report
This button switches you to the job report tool.

documents

This section shows all attached documents and allows new uploads via +new.

history

This section shows all changes made to the ticket.

NOTES
  1. You can add as many tickets as needed.
  2. Job status codes are set according to your needs. See here for more information.
  3. Tickets can be reassigned — but only to currently active users.
  4. Billable hours are handled separately on the +billing view.
