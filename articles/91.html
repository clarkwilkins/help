This is an example of processing an exchange item received from the time it's received until it's sent out for repairs. The example involves the use of our accountd platform for PO generation. You can substitute your own accounting system for these steps.

receiving the exchange item

Here, the exchange is received and linked to the original item. Status flags are set on the order that reflect the exchange processing.

  1. Receive and inspect the item. You're going to want to note any damage when the item record is created below.
  2. Add a new identity sticker.1
  3. Search orders, filtered by the client that returned the item, to get the order number. In the search results, click on the return link for the item originally sold on this order.2
  4. Create the exchange item.
    • In the return record, select +new item to open fields for a new exchange item.
    • Verify the part number matches your return item or adjust if needed.3
    • Set the new item's location.
    • Leave the condition as “customer exchange item” for now.
    • Add this item.4,5
  5. Switch to the new item6 and update the serial number.
  6. Set the exchange status
  7. Add photographs of the serial number and of the overall item itself. Be sure to include documentation of any anomalies.7
  8. Switch to the original order and update the order status. If other items are still outstanding on this order, set the status to either return items requested, if not overdue, or one or more items are overdue.
  9. Also on the order, check billing status. If more items are outstanding, the status should be billed,additional charges may apply. If all items are now returned, and you don't intend to levy late fees, you might set it to billed, no additional fees.

setting up the repair

Here, a cost source is set up to track repair costs which define the cost of the item. A repair job is set up and the item is sent for repairs. There's extensive cross-linking here as the item and repair job are tightly coupled.

  1. Switch to your accounting and generate POs for two expenses: the pending repair, and the initial shipment to the repair facility.8
  2. Switch to the inventory item and start a new cost source.
    • cost source type: new source job
    • new source name: Repair of item NNNN9
    • new source value: 010
  3. Create a new repair job for the item.
    • type: repair
    • priority: as needed
    • notes: “Repair of item NNNN”9
    After creating the job:
    • status: need to send the item to the vendor
    • vendor: as needed
    • purchase order: as defined in step 1
    • You may also want to set a followup with the vendor now, perhaps to verify arrival or discuss the repairs.
    • If relevant, add notes to the repair job about the pending repair. Cross-link the notes to the inventory item.
  4. Switch back to the item, and then to the cost source.
  5. It's time to add two future costs we already know are part of the source job.
    • Use +cost to add a $0 cost for "repair of item NNNN, job JJJJ" and reference the PO from step 1.
    • Use +cost to add a $0 cost for "shipment of item NNNN, job JJJJ to the vendor" and reference the other PO from step 1.
  6. In your accounting, go back to the original POs from step 1 and add a note referencing the repair job number.
  7. Add any additional repair paperwork and ship the item for repairs.
  8. Update the repair job status to item is in transit to vendor.
  9. Update the repair item to status sent for repairs.
  10. Make a note on the repair item with the shipping information. After creating it, use the notes editor and link the note to the repair job as well.
  11. If you included a repair form in step 7, upload a copy of it to the item, and add an additional link to the repair job.

At this point, you have created an exchange item and sent it for repair, while supplying lots of cross-referenced information that helps all team members to track and understand the process.
