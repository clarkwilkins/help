The purpose of this tool is to generate a compact tab-delimited file containing the transactions data for a single entity1 for a fiscal year.2

The remap statistics show three things:

GAAP
The remapped account that overwrites your internal accounting. This is setup in expense types, and it only affects overhead costs and “other” expenses. All transactions posting to COGS accounts are not affected.
expense type
The expense type as defined in your setup.
sum
The total payments on these expenses in the fiscal year.3

The table source-batch assignment is just a reference tool.

The actual report is source-batch export which has seven columns.

source
The mapping from the transaction type (journal entry, etc.) to a source-batch code.
id
The transaction's serial number.4
sum
The actual posting date of the transaction.3
account
The account where the transaction is posting.5
amount
The amount on one transaction line.
type
The transaction type.
payee
The transaction payee where applicable.6

The report runs immediately upon selection of an entity and is emailed to you in seconds. You can view it on the screen of course, but the email attachment is the same file and can open directly into a spreadsheet.7

NOTES

  1. If your accounting only involves one entity, the selection logic is not shown and this is handled “behind the scenes”.
  2. For each entity, an analysis is done and the years are determined by the presence of actual data, so you don't get to choose a year with nothing in it. Fiscal years are assumed to run from January 1 to December 31.
  3. accountd uses the actual payment date (as shown on your account statements. This is why you want to correct the posting date on charges and checks that may clear days (or even longer) after you initially create them — the cost can potentially flow into a new fiscal year.
  4. Every transaction is a set of at least one debit and one credit line — it can be more of either, as long as the total debits = total credits.
  5. On the displayed report, any transaction line gets remapped is noted with a “*”. This is not on the exported version.
  6. For expenses, the payee is taken from the vendor field, so when this is left empty, it's marked as “not set on expense XXX”. For checks, we always have the payee.
  7. The attachment is tab-delimited.
