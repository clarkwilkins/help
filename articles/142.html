This analysis shows how many times each recorded SKU has been been requested in one of four ranges: the last month, last three months,1 last six months, and the last year. The range options are on the right side just above the results table.

Above the table, you can see the total number of parts requests ever recorded, and how many are included in the selected timeframe.

SKUs which are not currently in-stock2 are set to gray and lined-out.

sku
This shows the part number, and also has the most recent request date below it.
name
The standardized part name for this SKU.
requests
How many times this SKU has been requested in the reporting period.

You can get a graphical analysis of the requests by clicking on any line. A chart will load showing each day in the reporting period and a bar indicator of how many requests for this SKU were logged.

NOTES
  1. The default report option is to show the last three months.
  2. The stock level indicated as “available” but does not differentiate based on the current item condition.
