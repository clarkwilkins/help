Notifications are best understood as a note linked to a specific item and targeted to a specific user. A notification differs from a general note in several ways:

notification
general note
Only links to one data object.
Can link to an unlimited number of data objects.
Targeted to a single user with option to notify them by email, SMS, or website only.
No direct notification of a user is available.
The target user can acknowledge the notification and add a reply.
Any user can add a followup note, but there is no direct linkage.
Visible on all platforms the target user can access (stockd, servicd, etc...).
Only visible on the connected item(s) in a single platform.

Creating a notification

  1. In the target item, click +new in the notifications section.
  2. Select your target user.
  3. Enter the notification text. The first line becomes the subject. Any following lines become the comments.
  4. Select an alert method if you want the target user to receive an email or SMS. In any case, the notification icon on the top menu will indicate an active statusas shown below.

Accessing the notifications manager

You can access your notifications by clicking on the manager icon on the top menu, right side. The icon has two states:

one or more active notifications: . In this state, the manager will show all notifications which are active (including acknowledged items).1, 2

no active notifications: In this state, the manager is accessible, but will not show anything.

Clearing notifications

The guiding principle here should be to not clear a notification if it includes a “call to action” that has not been accomplished. For this reason, the state “acknowledged, remains open” can be used to indicate this has been seen, but remains active.

The targeted user can also “respond” to the originator with a comment that becomes part of the public record. Comments can only be made or updated by the target user.

NOTES
  1. Once a notification has been closed, it will no longer be accessible here, but does remain permanently connected to the targeted item.
  2. When a notification is “acknowledged, remains open” it indicates the targeted user has seen it, but does not want to close it out yet.
