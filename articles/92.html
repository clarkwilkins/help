This is an example of handling a parts request in three scenarios: (1) you have the item available, and (2) you don't have it, but it has been requested before, and (3) you don't have it and it's never been requested before.

  1. Open the ☰ menu and select parts requests. The default view is for a new request.
  2. Select the customer account.1
  3. Enter the part number and let the search execute. Then execute the appropriate scenario below.

scenario 1: the part is in stock

Go to the top of the screen and click quote. You need to make a new sales quote, which will file a parts request automatically. You're done here.

scenario 2: the part is not in stock, but has been requested before

In this case, you will see a result under matching SKUs along with status from the last request, and stock counts, if applicable.

  1. Click on the matching result to load the part name. If you want to change it, feel free.2
  2. Change the status as desired.
  3. Add notes if desired.
  4. Save the parts request.

scenario 3: the part is not in stock, and has never been requested before

  1. Add the part name that will be associated with this SKU.3
  2. Change the status as desired. Your most likely choice is “never has been in stock”.
  3. Add notes if desired.
  4. Save the parts request.
NOTES
  1. If you don't see the customer/client on this list, click +new client and add them first, before continuing.
  2. For example, if the last person to set the name associated with this SKU included a typo. Changes made here will update all parts requests with the same SKU.
  3. Your entry sku:name will be saved and reused the next time this part number is requested.
