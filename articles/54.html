When selling items on exchange, it becomes critical to track the expected returns. stockd does this with “return records” which track the process.

order link
Every return record is directly linked to the sales order that originated it.1
item link
This is the original item sold to the customer.
return due
This is the due date for the exchange item.2
followup
An optional date selector for setting a followup on the return.
exchange item
When an exchange is processed, it should be linked here. This provides a historical record that you sold item A and received item B in exchange. If you need to put B into inventory first, use the new item option (discussed below).
received
Sets the date the exchange item was received.
exchange status
A selector that indicates the state of the exchange process.

The returns record allows you to directly add the received item to inventory without changing screens.

item number
Add a new (unused) item number here.3
part number/sku
The new item's part number.4
item name
The new item's part name.4
location
The new item's current location.
condition
The new item's current condition.

You can also send return notices from this tool. The notices are pre-formatted by stockd and available for +email (send as attachment with custom text in the email message), snf +pdf (generate a PDF for use).

Notes are added and edited with the standard interface. It's good practice to link the notes to the exchange and/or original items as well.

NOTES
  1. If more than one item is sold on exchange basis on an order, a separate return record is generated for each item, but they will all link back to the same originating order.
  2. The default number of days allowed for the exchange window is set in other tools: setup.
  3. The default set here is the first available item number.
  4. Usually, this is the same as the original item, so it's set by default.
