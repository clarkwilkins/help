The general search on the top menu bar is inherently limited in scope of search. For purposes of analyis, this tool offers a far more comprehensive set of search options. All filter controls here trigger a search change as they are modified. The default search result is all expenses in the last month.

Remember these filters work together and only return results matching all filters.
filter
effect
entity1
This control, which must be set, restricts the scope of the search to a specific entity. It filters the lists for expense types, posting accounts, credit cards, and vendors.
start/end
The beginning and ending dates for the search.2
(expense) type
Includes only expenses using the selected expense type.3
id/value/description
A number here will trigger matches on expense ID or expense amount. Any value will return a wild-card string match of the expense notes content.4

Entering two numbers as x...y where y >= x will return all expenses where the amount is in the range from x to y.

Entering two numbers as x:y where y >= x will return all expenses with IDs in the range from x to y.
credit card
Includes only expenses using the selected credit card.3
pending
Includes only expenses matching the selected pending state (don't care/yes/no).
posting account
Includes only expenses marked to post to the selected account.3
user
Includes only expenses which are either assigned to or payable to the user.
vendor
Includes only expenses which are either assigned to or payable to the vendor.3

It's also important to remember that, unlike general search, all results here are constrained to the selected entity.

NOTES
  1. This control only appears when more that one company/entity is defined.
  2. The default is for the last month to today.
  3. These lists are filtered and only show values actually in user.
  4. Entering “test” will return matches where the expense notes contain the words “Tested”, “attest”, “retesting”, etc.
