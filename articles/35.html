Simplexable operates three platforms1 that can share customer and user resources. This means you manage these records in a single “global” tool, and any changes you make are available to all three platforms. The data is extensible and shared as set by the “grant access” options. If you change a client from one platform, it's also changed for any others that use the client.

Companies act as containers

In servicd and stockd, you are running a single company. Your accounts can be shared within all three platforms, but servicd and stockd can only share customers within the same company ID. The accountd software, on the other hand, can manage the accounting for N > 1 companies at the same time, so it can access any client in any company designated in your group.2

Client records can store unlimited information in one of four sections.

basic information
customer name
This is a required field. It's usually good practice to make this field unique if possible.
account number
This value must be unique within a specific company/entity. More details are here.
status
The client can be designated as active (normal), suspended3, or inactive.
accountd
The client is accessible in normal accountd operations when this is checked.4
servicd
The client is accessible in normal servicd operations when this is checked.4
stockd
The client is accessible in normal stockd operations when this is checked.4
notes
Notes about the client are optional, and may be visible to regular users depending on the tool.
tags5
Tags about the client are optional, and may be visible to regular users depending on the tool.

locations

A client have an unlimited number of locations. To add a new location, click “new location”. Any existing locations can be editing by clicking on the location name in the list. For more information on locatons, click here

contacts

A client have an unlimited number of contacts. To add a new contact, click “new contact”. Any existing contacts can be editing by clicking on the contact name in the list. This is discussed in more detail here.

email

For each location, the software builds out a map of default contact-email pairs for each service. You can use this to make defaults for the client as a whole, and override them with sets for specific locations. This is discussed in more detail here.


NOTES
  1. servicd for service job management, stockd for warehouse/parts management, and accountd for multi-company accounting.
  2. Here's a real-world example. Simplexable runs servicd and accountd for our software operations. We run stockd and accountd for our parts sales operations. These are two separate companies, and they do not share clients between companies, but they do share them between platforms. The accountd software needs to be able to see clients from the company it's currently working in, so it can do things like search for invoices by customer, or print statements.
  3. At this time, there's no functional difference between suspended and inactive. These clients cannot be used in new transactions.
  4. All clients are accessible within a company ID (servicd, stockd) or an entities group (accountd) if this is checked. However, a user with access to manage clients can still see these groups within customer management, because this tool exists above the individual platforms.
