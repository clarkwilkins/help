In general, it's good practice to lock expenses once a transaction has been reconciled that connects to this expense. To be clear, accountd is not going to let you modify things that will break the reconciliation directly, but locking expenses — preventing further changes on them — is a safety measure to “freeze” changes on them.

The interaction is simple. Just pick an entity and click the button if any candidates for locks are found. A candidate is an expense which either has no locks at all, or the highest existing lock was set by a user at your level or lower.1 Clicking the button will apply your lock to all expenses on the list. The page reloads on success.

NOTES
  1. If user A creates a lock, it impacts user B when B has the same access level as A or a lower level than A.
