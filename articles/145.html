Email setup is used to set the “reply-to” parameter on emails sent from your platform.1 All emails are sent from the Simplexable API, and as such, they need a verified address, so we use noreply@simplexable.com.2 However, the API sets a “reply-to” header, so any replies will come to you by default.3

The purpose of email setup is to set a standardized reply-to address such as “accounting@thiscompany.com”. When this parameter is set for a platform1, the “reply-to” header is always set to this value, and the appended text is not used.

To add addresses to your setup, click +list. Add as many as desired, one per line, on the desired platform. Once you save the list, you can adjust the sender control to the desired sender (and save your changes again).

When you set a sender value, it's used for all email sent from the platform — regardless of the user involved. If the list is empty or no sender is selected, the default setting is applied.3

NOTES
  1. Platforms currently supported are accountd, servicd, and stockd.
  2. The verified address technique incorporates DKIM authentication to help us avoid having the email become tagged as “spam”.
  3. The default reply address is your registered email address in this account. When the default is used, the email will be appended with the text “(this email was sent by [your name])”.
