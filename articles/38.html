Any customer account can store an unlimited number of locations. Each location has information specific to it alone about the location address, notes, and billing information.

location name
This is a required field. It's usually good practice to make this field unique if possible.
billing address
This is an indicator field only used for characterization.
shipping address
This is an indicator field only used for characterization.
grant access to accountd
The location is accessible in normal accountd operations.1
grant access to servicd
The location is accessible in normal servicd operations.1
grant access to stockd
The location is accessible in normal stockd operations.1
notes3
Notes about the location are optional, and may be visible to regular users depending on the tool.
tags2,3
Tags about the location are optional, and may be visible to regular users depending on the tool.
address3
Holds optional address details for this location.
billing3
Holds optional billing details for this location.

NOTES
  1. All clients are accessible within a company ID (servicd, stockd) or an entities group (accountd) if this is checked. However, a user with access to manage clients can still see these groups within customer management, because this tool exists above the individual platforms.
  2. Tags are intended for a future expansion of capabilities, but you can start using them now.
  3. These fields are only shown on locations that have already been created.
