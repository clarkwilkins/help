Primary settings describe the user's fundamentals: name, contact, etc. This is also where an administrator can assign a new user login for any user. Fields in bold are mandatory.

name
The name of this user.
email
This is the address that the user can use for login validation, and where they will receive login information for their available platforms.1
phone
This phone is used for application-based messaging. It's also allowed for login validation.
active
Is the user allowed to access services? This affects all available services in your group.
user level
This setting is the master level for the user record. It's used to mediate whether a user A can access/modify a record for a user B. This is allowed in all cases where A has the same or higher level than B.
allowed to manage customers
Is the user allowed to manage customer records?
allowed to manage users
Is the user allowed to manage users records? If allowed, the user can access records for all users with a master level up to their own setting.
automatic response
Many text fields trigger when you type something into them. The response time before the trigger runs is set independently for keyboard based devices (laptops/PCs) and mobile devices (phones/tablets). The default is 1 second for keyboards and 2.5 seconds for mobile devices. The minimum allowable value is 1 second.
optional notes
Optional notes about the user.
NOTES
  1. This address is also used by default in all service configurations.
