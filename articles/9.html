This tool displays the actual reconciled balance for each recorded statement and the stored statement balance. Normally these will always match, and when they don't, you have a clear indicator that account reconciliation has been damaged. If the statement balance has not been stored yet, and you know it does match the bank/credit statement, click on the actual balance number to store it.

The actual balance is the reconciled amount from the prior month plus all amounts reconciled on this statement. This valus is saved into a special table as a static amount that can be used to detect changes to reconciled accounts.

This test is critical because it detects actions that have adjusted register balances that should be matching the statements. The situation usually occurs if someone does an adjustment in transaction analyzer that affects a reconciled line.

What to do when the amounts don't match

If you find a situation where your actual balance (column 2) does not match the statement balance, you have some troubleshooting to do. What's happened is that there are one or more transactions that are either missing or incorrectly posted.

The most efficient way to check this is to leverage the account reconciler.

  • Open tools:account reconciliation.
  • Set the account and use the statement date where you are having a problem.
  • Look for transactions that appear here.

A transaction here — especially one from a statement or more back — is probably needing repair. If the transaction does appear on the statement, you need to repair it with the transaction analyzer.

Fixing a statement line requires opening the transaction and restoring cleared to “yes”, and setting the statement date field to the one we're working on here. Check your work by running the reconciliation analysis again to see if the results match now.
