On the login screen, you are presented two options.

The easiest method is to enter your registered email address or mobile phone number. The system will look up your user record and send you an email with one (or more)1 login links. These links are single-click logins which have embedded authorization and do not require additional login actions.

Your second option requires your account to already have a username and password defined. In this case, you can enter these parameters, and, upon verification, you are logged into the software. If you choose to create this login, it's a very good idea to avoid common passwords for your own security. You might also take a look here for a method of dramatically increasing your security.

Unfortunately, errors do happen sometimes. The following is a list of problems this module detects, and solutions advice.

1026: invalid lookup data
This happens the email or phone number entered does not return a match.
1027: inactive account
The server found your user record, but it's set to inactive.

If you have another user on the installation who has access to user management, ask them to reactivate your account. If this is not an option, contact us via support@simplexable.com, and we will fix this for you — pending possibie authorization from a verified user at your company.2
1028: invalid recovery key
The server found a match for the username/passphrase you sent, but this record does not point to a valid user record. The key was deleted, and it will nto cause this error again.
1029: invalid email
The server found your user record, but the recovery email account is not valid.

If you have another user on the installation who has access to user management, ask them to reset your accouns email field. If this is not an option, contact us via support@simplexable.com, and we will fix this for you — pending possibie authorization from a verified user at your company.2
1030: email server error
This is a problem internal to our servers involving the format of the email being sent.3
1031: bad URI
This can happen if the web page address used to retreve the user record is compromised. Try making another selection on choose settings area.
1032: new login key was not installed
This is an internal failure during the process of changing the username/password for login.3
1033: old login key was not removed
This is an internal failure during the process of changing the username/password for login.3
1034: login reference was not updated
This is an internal failure during the process of changing the username/password for login.3
